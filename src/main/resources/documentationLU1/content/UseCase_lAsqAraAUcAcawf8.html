    • Genruiker voert de invoervelden in (Naam, Email, wachtwoord, KVKnummer, VAT nummer,Naam bedrijf, adres, plaats.
    • Gebruiker drukt op verzenden.
■ Business Logic
    • // Write down what user expect the system to react upon certain condition (e.g. low inventory alert level)
    • systeem kijkt op er al een gebruiker bekend is op het device (file check op device)
    • systeem controleert of ingevoerde email al in mogelijk aanwezige file staat.
    • gegevens die door de gebruiker zijn ingevoerd worden in een file opgeslagen.
    • Wachtwoord wordt hierbij encrypted opgeslagen
    • file met deze gegevens wordt lokaal opgeslagen

Scenario
Steps
1. Gebruiker start applicatie
Gebruiker voert voornaam, achternaam, adres, bedrijfsnaam, telefoonnummer, VATnummer, KVKnummer in
2. Gebruiker voert voornaam, achternaam, adres, bedrijfsnaam, telefoonnummer, VATnummer, KVKnummer in
Gebruiker voert voornaam, achternaam, adres, bedrijfsnaam, telefoonnummer, VATnummer, KVKnummer in2
3. Gebruiker drukt op verzenden
Gebruiker voert voornaam, achternaam, adres, bedrijfsnaam, telefoonnummer, VATnummer, KVKnummer in2
4. SYSTEM slaat de gegevens op in ClientRegistrationDAO
Gebruiker voert voornaam, achternaam, adres, bedrijfsnaam, telefoonnummer, VATnummer, KVKnummer in
5. if database give error
    5.1. SYSTEM "Gegevens konden niet worden opgeslagen in de database"
   end if
6. SYSTEM "Gegevens zijn correct opgeslagen in de database"
